Changelog

List of data relevant changes in each patch. These notes are not guaranteed to be comprehensive.

Known Issues

A list of known issues relevant to data and modding.

  • For Reign of the Warlock mode games, skills are not included in staff mods unless their $!skills#reqlevel!$ is exactly one of the following: {1, 6, 12, 18, 24, 30}
  • Monster levels for the Bind Demon skill are restricted to 127 when saving into save data
  • Non-zero $!desecratedzones#behavior!$ values do not work with $!desecratedzones#difficulty-unique_mod_chance_boosts!$. The functionality does work with $!desecratedzones#herald-unique_mod_chance_boosts!$
  • $!desecratedzones#difficulty_scale!$ functionality does not work

Note for modders: Report bugs, regressions in behavior, or crashes to the Diablo II: Resurrected bug report forum. Include "[Modding]" in the title of your report. Tips:

  • If possible, include offending rows of data. For example, the cubemain line the issues appears in
  • For crashes, include detailed repro steps as well as the crash id found in the crash window

Reported issues are not guaranteed to get tracked. Tracked issues are not guaranteed to appear on this list. Issues on this list are not guaranteed to have a fix version.

3.1.2

  • Marked the new Data Guide as not WIP!
  • Fixed a regression where $!cubemain!$ recipes that output unique items would have unexpected item levels
  • Fixed a regression where monster levels were restricted to an int8
  • Mods missing the "DataVersionBuild.txt" file will now prompt the out of date data prompt on startup. Modders should include this file in their mods to prevent the prompt from appearing. Located at /data/global/DataVersionBuild.txt
  • Updated $!monstats#SplEndDeath!$
