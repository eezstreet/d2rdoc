About

Community note: This page was authored by Blizzard Entertainment and is presented here unaltered.

The Data Guide is an html-format documentation of all-things data in Diablo II: Resurrected. It is used by both developers and community modders. The guide displays the documentation as html files. Simply click on them to open them in your web browser.

Generated .html files

In the /files/ folder are .html files for every page to display in the guide. These are generated automatically using scripts and consume the .js files in the /data/files/ folder.

You might notice that these files load the .js using script tags instead of file loading (aka a CORS request). This is because browsers no longer let you run CORS requests without a server host. To allow users to just open each .html file directly, we load everything we need explicitly using script tags. Read more here.

.js files

In the /data/files/ folder are .js files for every data file the game consumes and more. These .js files contain a json object that stores the documentation.

Adding a new file or field

  1. Add your file or field to c++ code.
  2. Run "Update Bins" in the Launcher. This will generate a new /data/DataGuideFieldExport.json file. This file is an export of what fields the code expects as well as its memory when reading and storing.
    • Make sure to build Release Assert before doing this so that the code that runs "Update Bins" will know about your change. Otherwise /data/DataGuideFieldExport.json will not be updated.
  3. Run /python/update_fileguide.bat. This will generate the boilerplate for your file or field.
    • This script requires python 3 as well as "py -3" to be set up in your system's PATH. If this is not the case you can manually run the script using /python/update_fileguide.py with the args -a all -data [path to /data/ folder] -html [path to /files/ folder] -p
  4. If you're adding a file:
    1. The corresponding .js file has been created by the script and placed in the /data/files/ folder.
    2. Manually add your file to the side index in /data/siteIndex.js.
    3. Fill out the .js file.
  5. If you're adding a field:
    1. The field has been added to the corresponding .js file in the /data/files/ folder.
    2. Locate the field in the .js file, move it to its correct location, and fill it out.
  6. Afterwards, run the python script from step 2 again, using the same arguments.
    • This will ensure the search index is updated.
    • Check for any failed validation in the output. If there is some, fix it then repeat the command until there are no validation errors.
    • To run just the validation, run /python/validate_fileguide.bat. This is only recommended if you are just changing descriptions since changing many things, like field names, require the full update to run to update things like the search index.
  7. Add a note to the changelog.
  8. You're good to submit!

Editing an existing file or field

  1. Find the corresponding .js file in the /data/files/ folder.
  2. Edit what you need to in the file.
  3. Run /python/update_fileguide.bat. This will ensure things like the search index is up to date.
    • This script requires python 3 as well as "py -3" to be set up in your system's PATH. If this is not the case you can manually run the script using /python/update_fileguide.py with the args -a all -data [path to /data/ folder] -html [path to /files/ folder] -p
    • Check for any failed validation in the output. If there is some, fix it then repeat the command until there are no validation errors.
    • To run just the validation, run /python/validate_fileguide.bat. This is only recommended if you are just changing descriptions since changing many things, like field names, require the full update to run to update things like the search index.
  4. Add a note to the changelog.
  5. You're good to submit!

Removing an existing file or field

  1. Remove your file or field from c++ code.
  2. Run "Update Bins" in the Launcher. This will generate a new /data/DataGuideFieldExport.json file. This file is an export of what fields the code expects.
    • Make sure to build Release Assert before doing this so that the code that runs "Update Bins" will know about your change. Otherwise /data/DataGuideFieldExport.json will not be updated.
  3. If you're removing a file:
    1. The corresponding .js file can be found in the /data/files/ folder. Mark the file for deletion in perforce.
    2. The corresponding .html file can be found in the /files/ folder. Mark the file for deletion in perforce.
    3. Remove your file from the side index in /data/siteIndex.js.
  4. If you're removing a field:
    1. The corresponding .js file can be found in the /data/files/ folder. Delete the field from the file. Make sure to not leave any trailing commas.
  5. Run /python/update_fileguide.bat. This will update the search index with your removals.
    • This script requires python 3 as well as "py -3" to be set up in your system's PATH. If this is not the case you can manually run the script using /python/update_fileguide.py with the args -a all -data [path to /data/ folder] -html [path to /files/ folder] -p
    • Check for any failed validation in the output. If there is some, fix it then repeat the command until there are no validation errors.
    • To run just the validation, run /python/validate_fileguide.bat. This is only recommended if you are just changing descriptions since changing many things, like field names, require the full update to run to update things like the search index.
  6. Add a note to the changelog.
  7. You're good to submit!

Reference links

In almost all areas that accept text to display, like a file's "overview" or a field's "description", a reference link may be placed. This is a shorthand for creating an html link to another page or element. All links are opened in a new tab. This is useful for linking to other files and fields throughout the data guide. The format is as follows:

  • $!page!$ - Links to another page.
  • $!page#element!$ - Links to a specific element in another page.
  • $!#element!$ - Links to a specific element in the current page.

Note: You do not need to place the referenced file in referenceFiles for these links to work.

Considerations

This fileguide is shipped with the game in game data. It is important to consider a few things:

  • Everything in this data guide is public, from a field's description to comments in the python code.
  • Media can bloat the size of websites. Don't add any, like images, unless absolutely necessary.
